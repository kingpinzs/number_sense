## What Didn't Go Well

### 1. **Single-Pass Implementation** (CRITICAL)
Dev agents completed tasks linearly without iterative verification. They implemented, tests passed, marked done - but didn't circle back to verify integrated behavior.

**Evidence:**
- Story 4.2: Magic Minute timer rendered but showed NO challenges (0 correct the whole time)
- Story 4.2: Duration calculation was wrong - obvious on visual inspection
- Story 4.4: Missing compound index - would fail on first DB query

**Root Cause:** No enforcement of verification loops in dev workflow.

### 2. **Deferred Test Coverage**
Story 4.3 code review flagged missing RTL component tests for MicroNumberLineDrill, MicroSpatialDrill, and MicroMathDrill as HIGH severity. These were marked as "AI-Review follow-ups" and never completed.

### 3. **Session ID Inconsistency**
Multiple stories had confusion between string UUIDs and numeric Dexie auto-increment IDs. Story 4.2 had parseInt returning NaN. A TODO was added but never addressed.

### 4. **Type Definition Duplication**
Stories 4.2 and 4.4 both had duplicate type definition issues that had to be fixed in review.

> **"Agents need to loop over their work multiple times after they think they're done. Triple-check, not single-pass."**

This insight addresses the root cause of most issues found in Epic 4.

### Triple-Check Protocol (NEW)

1. **First pass:** Implement feature, write tests, tests pass
2. **Second pass:** Run `npm run dev`, manually use the feature, verify it works visually
3. **Third pass:** Check edge cases - empty data, wrong inputs, timeouts
