## What Didn't Go Well

### 1. **Integration Gaps in Story 2.1** (CRITICAL)
**Issue:** Assessment wizard component was marked 'done' but was not routed to `/assessment`. The feature was not accessible in the running application.

**Impact:** Story appeared complete based on test coverage, but integration was non-functional until manually fixed.

**Root Cause:**
- Definition of Done didn't require manual verification in running dev server
- Code review focused on test coverage and code quality, not functional verification
- Acceptance criteria didn't explicitly require routing verification

### 2. **Broken Questions in Story 2.3** (CRITICAL)
**Issue:** Spatial awareness questions had functional issues despite passing all unit tests.

**Impact:** Tests provided false confidence - questions didn't work correctly in the actual application.

**Root Cause:**
- Unit tests verified components in isolation but not actual functionality
- No manual testing in running application as part of DoD

### 3. **E2E Test Required Pre-Fixes** (CRITICAL)
**Issue:** Story 2.7 (E2E test) only passed after Project Lead manually fixed routing and question issues from Stories 2.1 and 2.3.

**Impact:** E2E test should have failed immediately, exposing gaps in earlier stories' verification processes.

**Root Cause:**
- Integration testing was deferred to Story 2.7 instead of happening incrementally per story
- Component stories didn't verify their own integration points

### 4. **Process Philosophy Gap** (CRITICAL)
**Issue:** E2E test was treated as the integration verification step rather than the final guarantee.

**Expected:** Each story verifies its own integration; E2E test should pass on first run if all component stories are properly verified.

**Actual:** Integration issues weren't caught until E2E test, requiring manual intervention.

## Retrospective Learnings

### Key Insight:
**Test coverage is not a substitute for functional verification.** A story can have 100% test coverage and still not work in the application if tests don't verify integration points (routing, UI rendering, data flow).

### Process Change:
**Incremental integration verification > Deferred E2E testing**

Each story must verify its own integration. The E2E test at the end of an epic is the final guarantee, not the primary integration verification.

### Cultural Change:
**"Done" means "works in the browser"**, not "tests pass"
