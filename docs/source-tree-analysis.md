# Source Tree Analysis

## Top-Level Layout

```
.
├── index.html                 # Static entry point, tab markup, script/style tags
├── scripts/
│   └── main.js                # All application logic (tabs, wizards, training, analytics)
├── styles/
│   └── styles.css             # Theme, responsive layout, mini-game visuals
├── docs/
│   ├── index.md               # Documentation index (generated)
│   ├── project-overview.md    # Executive summary
│   ├── source-tree-analysis.md# This file
│   ├── architecture.md        # Runtime/module architecture
│   ├── component-inventory.md # Component catalog
│   └── development-guide.md   # Setup and verification playbook
└── bmad/
    └── bmm/                   # BMAD workflow assets used to generate documentation
```

Ancillary folders such as `.vscode`, `.claude`, and `.gemini` store editor/agent configuration and do not affect runtime behavior.

## Directory Details

- `index.html`
  - Declares five sections (`assessment`, `coach`, `train`, `cognition`, `progress`) plus a wizard overlay container.
  - Loads Chart.js and the date-fns adapter from CDN before `scripts/main.js`.

- `scripts/`
  - `main.js` is segmented into comment-delimited regions: tab navigation, persistent storage helpers, wizard utilities, inline training modules, assessment/coach/cognition flows, and progress analytics.
  - There is no module bundler; everything executes in the global scope once the script tag loads.

- `styles/`
  - `styles.css` defines the entire design system (CSS custom properties, cards, tab buttons, wizard layout, dot grids, keypad, number line, and responsive tweaks).
  - Media queries at 720 px and 520 px keep the wizard usable on tablets and phones.

- `docs/`
  - Houses this generated documentation set and the workflow status file (`bmm-workflow-status.yaml`) tracking BMAD progress.

- `bmad/`
  - Contains BMAD configuration (`bmm/config.yaml`), workflow templates (`bmm/workflows/**`), and supporting docs.
  - These files orchestrate documentation workflows but are not loaded by the Number Sense Suite at runtime.

## Entry Points and Assets

- **HTML:** `index.html` is the sole HTML document; all sections are initially rendered and toggled via JavaScript.
- **JavaScript:** `scripts/main.js` runs immediately after Chart.js loads. There are no imports, so any new module must be included manually.
- **CSS:** `styles/styles.css` styles both the static sections and dynamic wizard content. No CSS preprocessor is used.
- **Static Assets:** There are no images or fonts bundled; graphics are created via CSS and canvas.

## Build and Deployment Notes

- Because the project is 100 % static, any HTTP file server or CDN can host it. No build step is required, but bundling/minifying could be added later if desired.
- Keep CDN references (Chart.js, adapter) pinned to known-good versions to avoid surprise regressions.
- When adding new documentation generated by BMAD workflows, store it under `docs/` to keep parity with `output_folder` in `bmad/bmm/config.yaml`.
